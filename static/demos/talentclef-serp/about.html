Hybrid Dense-Sparse Retrieval for TalentCLEF Task A

This demo accompanies the TalentCLEF 2026 Task A submission Hybrid Dense-Sparse Retrieval and Re-Ranking for Multilingual Resume Retrieval from Job Descriptions. Task A is a contextualized job-person matching task: given a full job description, the system ranks candidate resumes by relevance.

The interface shows a SERP-style version of the primary run: JobBERT first-stage retrieval followed by SPLADE-based reciprocal-rank-fusion re-ranking.

Task And Data

The retrieval problem is difficult because both job descriptions and resumes are long, structured, multilingual, and packed with evidence. A good ranker needs broad semantic matching, but also needs to respect concrete evidence such as tools, technologies, certifications, credentials, and role-specific skills.

  • Tracks shown by the demo: monolingual English-English and Spanish-Spanish retrieval.
  • Development split: 10 queries and 472 resumes per language.
  • Test split: 40 queries and 476 resumes per language.
  • Main evaluation metric: MAP, with nDCG reported as an additional ranking-quality measure.
Resume Representation

Each resume is represented by normalized text built from profile, experience, skills, education, certifications, languages, and remaining resume content. This keeps the pipeline reproducible while preserving evidence useful for candidate matching.

How The Approach Works

The submitted system is a two-stage ranker. The first stage finds semantically plausible candidates; the second stage reorders those candidates using lexical evidence that is especially important in hiring contexts.

1
Build resume and query text

Each CV is converted into a retrieval document by concatenating profile, experience, skills, education, certifications, languages, and remaining resume content. Job descriptions are normalized into query text.

2
Retrieve candidates with JobBERT

TechWolf/JobBERT-v3 encodes the job description and CVs into the same embedding space. This dense first stage captures broad role compatibility even when the job and resume use different wording.

3
Re-rank with SPLADE evidence

SPLADE adds a sparse neural signal over the JobBERT candidate set. This makes exact and expanded lexical evidence, such as tools, certifications, domain terms, and required skills, count more directly.

Final Ranking

The final submitted run combines the JobBERT candidate ranking and the SPLADE ranking with reciprocal-rank fusion. RRF rewards candidates that appear high in both rankings while still allowing either dense semantic evidence or sparse lexical evidence to move a strong candidate upward.

Systems Compared

The paper compares lexical, sparse, dense, and second-stage re-ranking systems. JobBERT is already a strong baseline, so the key question is whether a complementary re-ranking signal improves candidate ranking.

What The Results Suggest
  • SPLADE re-ranking gives the best MAP in both English and Spanish.
  • The gain comes from combining broad semantic matching with precise lexical evidence.
  • Specific requirements such as tools, technologies, certifications, and skills remain central in resume retrieval.
  • ColBERT is not the strongest overall system, but it is nearly tied with SPLADE in Spanish MAP.
  • ColBERT has the best Spanish nDCG@50, which suggests token-level late interaction is promising for multilingual resume ranking.
  • Standalone sparse retrieval and query-rewrite variants were less robust than the two-stage systems.
